Getting Started

Projects

Projects are spaces where users are able to create apps and models. Created projects can be found under the tab Projects in the sidebar. This tab contains all projects that you have created or have been granted access to.

Create a Project

To be able to create a project, users need to be logged in. Users can create a new Project by following the steps below:

  1. Click on the Projects tab shown on the sidebar. (Only shown if user is logged in.)
  2. Click the New Project button in the top right corner.
  3. Select the project template you want to use (currently there is only one template available).
  4. Enter project name (mandatory) and description (optional). Users can also change the project template here if they want.
  5. Click the Create Project button to create the new project.

This will take you to the project dashboard. Here users will be able to see an overview of the whole project. New tabs will be available on the sidebar as well. The categories for these tabs are explained below:

  • Objects: Model objects created in the current project.
  • Compute: Pre-configured data science environments for experimentation and training ML models. (Currently, this tab will only have a pre installed Jupyter lab instance available for the user. Using this instance, users can e.g can run experiments and also create model objects mentioned above. The tutorials below will elaborate on this.)
  • Serve: The most common serving applications used for deploying ML models as well as apps such as shiny and dash. (Custom applications can be added to this list if requested).
  • Settings: User can access and change the current project settings

Settings

The Settings page contains all the information about your Project and its components. Some of the things you can do there are:

  • Change the Project's description
  • Grant access to the project to registered users.
  • Transfer ownership of the project to another user.
  • Permanently delete the project.

Creating Apps and Resourcess

You can create resources by going into the Serve tab. Here you have the following options:

  • Shiny App
  • Dash App
  • Tensorflow Serve
  • PyTorch Serve
  • Python Model Serve (Using FastApi)

Shiny

This is a service to deploy a shiny app that has been packaged as a docker container and is available on docker hub. To deploy a Shiny App click the Create button on the Shiny App card in the Serve Tab. Then Enter the following information in the form:

  • Name: User specified name for the app (e.g SciLifeLab Shiny Example)
  • Subdomain: This is the subdomain that the deployed app will be available at. It is not mandatory to choose. By default a random subdomain (e.g r46b61563) is generated if no option is chosen here. Then the app would be available at r46b61563.serve.scilifelab.se. If a user wants to create a more suitable subdomain, they can do this by scrolling down to the form that contains the option to Add Subdomain. Here the user can create a subdomain of their choice (given that it is not already taken). Then, this subdomain will appear in the Subdomain options and the final domain will be user-created-subdomain.serve.scilifelab.se.
  • Permissions: Permissions for the project. There are 3 levels of permissions for an app:
    • Private: The app can only be accessed by the user that created the app. (Sign In required)
    • Project: All members of the project will be able to access the app. (Sign In required)
    • Private: Anyone that has access to the url can access the app.
  • Path to folder:This is the path to the folder (inside the docker container) in which the main shiny application file is located (usually app.R).
  • App Port: The port that the shiny server runs on (usually 3838).
  • DockerHub Image: Name of the image on DockerHub.
